Introduction

GENFIRE, for GENeralized Fourier Iterative REconstruction, is an open-sourced, GUI-enabled python package for 3D reconstruction from arbitrarily oriented projection images. GENFIRE was developed at the University of California, Los Angeles and has found successful applications in a number of imaging fields including Atomic Electron Tomography (AET), electron cryo-tomography, tomographic Scanning Transmission X-ray Microscopy (STXM), and phase-contrast X-ray ptychographic tomography.

GENFIRE iterates between real and reciprocal space using the Fast Fourier Transform (FFT), enforcing constraints in both. This technique of alternating projections onto convex sets allows GENFIRE to recover unmeasured datapoints, producing reconstructions with superior contrast and mitigated missing-wedge artifacts when compared with traditional tomographic techniques such as filtered backprojection. This type of reconstruction algorithm was originally demonstrated in the form of Equally Sloped Tomography (EST). GENFIRE builds upon EST by introducing a 3D gridding routine, allowing it to tolerate arbitrary Euler rotations of the input projection images. In addition, GENFIRE introduces a novel technique for enforcing Fourier constraints deemed resolution extension/suppression and implements a modified version of R-free from X-ray crystallography for the purposes of prevention of overfitting data.

GENFIRE is written in python using primarily NumPy and SciPy. It runs on python 3 and the GUI is programmed using PyQt5. The user interface was designed with Qt Creator. Embedded plots were made with matplotlib.

Coordinate Conventions

The coordinate and Euler angle conventions used in GENFIRE are the standard defined by Heymann et. al (2005).
For 3D arrays the fastest changing index represents X, the second-fastest represents Y, and the slowest Z. GENFIRE is built on top of NumPy and uses row-major array storage (sometimes called "C-style". As such an array is indexed as [x, y, z].
Euler rotations are composed of angle triplets in the form (phi, theta, psi) where:

  • Phi represents a rotation about the Z-axis
  • Theta represents a rotation about the new Y-axis
  • Psi represents a rotation about the new Z-axis
*Note* in the current implementation, array sizes must be even.

File Formats

GENFIRE natively works with .mrc, .mat, and .npy files for volume data. Euler angle triplets and error curves are stored as plain .txt files. File formats can be mix-and-matched using the GUI without trouble, or users can convert filetypes from the command line using GENFIRE's fileio module. For example the following code reads in an .mrc file and saves it in .mat format.

$ from GENFIRE.fileio import readVolume, writeVolume
$ volume = readVolume('foo.mrc')
$ writeVolume('bar.mat', volume)
